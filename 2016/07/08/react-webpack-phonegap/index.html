基于webpack+react+phonegap开发APP

关于webpack,react,phonegap,react-router,redux,weui,es6每一种技术文档资料都很多.你可以做个击破.这里我只讲到你下载了我的项目直接能跑起来本人亲试能跑起来,更深的层次自己去摸索吧.

生产开发

  1. npm install -g webpack webpack-dev-server cordova-hot-code-push-cli //-g 的东西要安装完
  2. git clone https://github.com/AbingManling/react_webpack_phonegap && cd react_webpack_phonegap
  3. npm install npm run dev //开发
    3.cd node_server && npm install && node app.js //nodejs jsonapi

发布

  1. 修改assets>index.html 引入cordova.js cordova热更新依赖cordova.js
    assets>js/router.jsx 这一行关闭DevTools = require(‘./handle/devtools’)
  2. npm run build 打包
    2.
    cd node_server/static && sudo cordova-hcp server
  3. 修改chcp.json和chcp.manifest两个文件 具体phonegap搭建请到http://www.phonegap100.com/ cordova热更新文档 弄不好看树根的视频

我的思路

以前搞过ionic开发app,最近这一段时间比较闲,利用工作和业余高了一套新的跨平台APP开发模式.
ionic组件很全,开发速度快,但它是重量级武器,用于移动端性能是它的瓶颈,
react是fecebook开源的一个前端V层框架,他采用单向数据流,虚拟DOM等思想提高性能,它是一个轻量级的框架,更适合移动端.
react问世不就fecebook有推出了react-native基于前端开发原生APP,现在android,ios支持,
它和phonegap的原理又不一样,phonegap就是封装webview并实现js和原生代码的相互调用,提供原生代码封装成API给js调用.
react-native的原理应该类似与cocos吧.关于这三个框架想了解更多,请去官网…
性能自然是react-native最好,但是听说react-native上手很难哦,关于我之前是搞APP学了那么多phonegap插件,现在是做前端,
我更愿意用react+phonegap开发app.
