打包规范

现在前端代码全面支持压缩了,压缩过后我们的核心js代码bootstrap.jsc从2.4M减少到1.1M.一方面商户后台的性能提高了,另一方面保护了前端代码的知识产权,保护了公司的资产.但是这里有两点需要注意,不然gulp压缩过后我们的代码报错或运行不起来.

  1. 我们的controllers,services注入一定不要简写

    如果控制台出现下面的报错说明你的controller,services代码不规范.

    正确的写法
    controllers

    services

    错误的写法
    controllers

    services

    这种写法在开发的过程能运行,打包后就报错了.当controller,services注入为空的时候打包后也能运行,但是一定要按规范来不要简写!!!

  2. status 路由的resolve参数不要用简写的形式.

    status 路由的resolve如果不规范,打包过后页面也不会报错,但是代码不能运行.

我的前端感悟

    曾经做过半年的java开发,现在从事前端行业一年半了.两年的程序员生涯感悟颇多.
    前端和后端还是很有区别的.前端强调用户体验,后端要求逻辑算法.一开始当我从事Java开发的时候很鄙视前端的,认为三五天就可以学完的,像linux命令行那样会一些简单的脚本语言就可以了,而且它没有面向对象.当我开始做前端的时候接触的是angulerjs我一下就蒙了,发现所有的东西都要重新学,发现js也有面向对象js有很多东西闭包,回调函数,原型,function,事件等很多东西第一次接触.再接着我做ionic开发,发现js好神奇还可以做android,ios app,好屌.渐渐的我懂了这一切都是nodejs催生的,不光是跨平台app而且还可以做桌面应用,java有单元测试,maven,js也有QUnit、Jasmine, gulp等;一时间我觉得javascript要统治世界了.又过了一段时间,我开始认识到性能问题,认识到每种语言,每种框架有它自身的特点,优点,缺点,这些决定了它的试用场景,比如nodejs单线程,非阻塞I/O,事件驱动三大特性决定它适合做考试系统,聊天室,用户表单收集,图文直播,提供JSON的API.
    老实说,自己也是比较拼的那一类程序员,英文不好,有些文档看不懂,我习惯去淘宝买教程,java的,前端的,php的,搞ionic还买过iOS的教程,当然一首资料(英文文档)是最重要的.想做一个全栈工程师,想写的代码受人尊重.这两年的感悟总结一下,希望自己以后进步的更快,分享给那些有为程序员.

1. 博学,见多识广.

当你踏上程序员这条路的时候,你对软件什么都不懂得时候,你只有通过学习学习.你学过java的基本对象,做一个成绩排序列表插入一个成绩你最先想到的是数组写一个方法排序数据当你学了java的集合之后,你就因该直接了当的new HashSet().记住尝试用你学过的新东西并记住它.这是最简单的方式.

2. 归纳总结,学习别人的优点.

开始做项目了,分你一个留言版模块,你去网上找了一些资料,改改实现了.这是时候,你就该记住表单怎么写,文件上传怎么写,java是这样处理的,记住其它语言也是这样处理的.你的组长看了看你的代码,留言板列表需要头像,而头像在member表里面,你写了两条sql,常常的代码,你的组长给你改成了一条,这时候就更应该归纳总结一下,
如果下次再让你做一个列表你该怎么写,类似的问题不能再让你的组长给你的代码了.渐渐的简单的增删改查你熟悉了,你可以看看你组长的代码怎么写的,权限模块,你的组长用一条sql搞定某个用户可以有那些菜单操作的权限.时刻记住归纳总结.

3. 思考问题,培养软件思维

其实你的大学学过很过软件相关的知识,貌似工作上都没用到,比如你学过软件测试,软件测试黑盒测试有个方法交边界值测试,软件过半的错误都在边界上,你写过过的代码,你就应该在这里考虑一下.思考事物之间的联系.当然思考不仅在此,人的一生使时时刻刻都在思考问题,而软件就是为了解决重复的劳动,重复的思考.你想挣更多的钱,你就的提高你的技能,别人晋升为组长,你和别人的差距在哪里?如何提高自己?人一定要有理想,你有理想了才可能会成功.当分给你了一个任务,你立刻就开始做了,而你的同事抽着显示屏貌似在发呆,而你推翻了一套又一套方案,这时候你的同事已经把它的任务做完了,你觉得你够努力,为什么不如别人?其实你和你同事之间只有思维方式不同,这就决定了他是优秀的,你是差的.你所推翻的那一套有一套方案他都思考过了.你纠结于css不能获取屏幕的高度,js不能操作iframe的内容,js读取不了文件,而他跟本不会考虑这些问题,这些违背软件设计的理论.这时候你因该吧软件功能的书拿过来读一读,软件危机,软件设计的原则.我记忆最深的是刚做前端的时候,面试官问我父子div都有一个onclick方法,哪一个先执行,你就要想到js作为一门程序设计语言发展了这么久,如果让你来设计,你就应该设置成父div的onclick可以先执行,子div的onclick也可以先执行,而且是可以设置拿个先执行,并设置默认值.

4. 谦虚包容别人,追求极致.

谦虚使人进步,包容别人,你才能融入集体,追求极致你才可能做出激动人心的产品,追求极致你代码写的到位,到位就是,简介,最优算法性能好,高內聚松耦合,体验好……追求极致要对自己严格要求写作的代码bug越少越好,代码要审查,上线后很多问题测试是测不出来的,因为需要白盒测试,所以你也过代码要就行单元测试.追求极致要求你永远都不要停止学习…

5. 善于使用工具.

人类之所以能在千千万万的动物中脱颖而出,成为大自然的看护者.因为用使用工具,替代身体上的器官来适应大自然.做软件开发的过程中很多问题你肉眼凡胎看不出来,用Beyond Compare吧! 像印象笔记,webstorm,eclipse等熟练的使用它们吧!

彻底解决mac下terminal路径显示问题

www.2cto.com
Bash代码
echo $PS1

这时会显示类似\h:\W \u\$这样的一段信息，这些信息就是用来定义提示符的显示方式，具体的细节下边会列出。
Bash代码
cd~
open -e .bash_profile

这时会打开TextEdit，按照你的要求，可以在里边输入
Bash代码
export PS1=”\u \w$”

保存退出。
如果没有.bash_profile文件的话就创建一下，终端输入

cd ~
touch .bash_profile

webpack+react+phonegap

基于webpack+react+phonegap开发APP

1. run起来

关于webpack,react,phonegap,react-router,redux,weui,es6每一种技术文档资料都很多.你可以做个击破.这里我只讲到你下载了我的项目直接能跑起来本人亲试能跑起来,更深的层次自己去摸索吧.

生产开发

  1. npm install -g webpack webpack-dev-server cordova-hot-code-push-cli //-g 的东西要安装完
  2. git clone https://github.com/AbingManling/react_webpack_phonegap && cd react_webpack_phonegap
  3. npm install npm run dev //开发
    3.cd node_server && npm install && node app.js //nodejs jsonapi

    assets>js/router.jsx 这一行关闭DevTools = require(‘./handle/devtools’)
  2. npm run build 打包
    2.
    cd node_server/static && sudo cordova-hcp server
  3. 修改chcp.json和chcp.manifest两个文件 具体phonegap搭建请到http://www.phonegap100.com/ cordova热更新文档 弄不好看树根的视频

我的思路

以前搞过ionic开发app,最近这一段时间比较闲,利用工作和业余高了一套新的跨平台APP开发模式.
ionic组件很全,开发速度快,但它是重量级武器,用于移动端性能是它的瓶颈,
react是fecebook开源的一个前端V层框架,他采用单向数据流,虚拟DOM等思想提高性能,它是一个轻量级的框架,更适合移动端.
react问世不就fecebook有推出了react-native基于前端开发原生APP,现在android,ios支持,
它和phonegap的原理又不一样,phonegap就是封装webview并实现js和原生代码的相互调用,提供原生代码封装成API给js调用.
react-native的原理应该类似与cocos吧.关于这三个框架想了解更多,请去官网…
性能自然是react-native最好,但是听说react-native上手很难哦,关于我之前是搞APP学了那么多phonegap插件,现在是做前端,
我更愿意用react+phonegap开发app.
